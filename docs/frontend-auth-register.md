# مستند فرانت‌اند: ثبت‌نام احراز هویت

این سند فقط قرارداد سمت فرانت برای ثبت‌نام کاربر در ماژول احراز هویت را توضیح می‌دهد.

## Endpoint

| مورد | مقدار |
| --- | --- |
| Method | `POST` |
| URL | `/api/Auth` |
| نیاز به توکن | ندارد |
| Content-Type | `application/json` |

> نکته: بعد از ثبت‌نام، API توکن برنمی‌گرداند. برای دریافت توکن باید پس از ثبت‌نام، کاربر از مسیر لاگین استفاده کند.

## Request Body

```json
{
  "firstName": "علی",
  "lastName": "رضایی",
  "phoneNumber": "09123456789",
  "passwordHash": "StrongPass123",
  "isCompleteProfile": false,
  "avatarImageName": null,
  "gender": 1,
  "birthDate": "1995-05-20T00:00:00"
}
```

### فیلدهای ورودی

| فیلد | نوع | اجباری | توضیح | اعتبارسنجی |
| --- | --- | --- | --- | --- |
| `firstName` | `string` | بله | نام کاربر | حداکثر ۱۰۰ کاراکتر |
| `lastName` | `string` | بله | نام خانوادگی کاربر | حداکثر ۱۰۰ کاراکتر |
| `phoneNumber` | `string` | بله | شماره موبایل کاربر | باید با `09` شروع شود و دقیقاً ۱۱ رقم باشد؛ مثال: `09123456789` |
| `passwordHash` | `string` | بله | رمز عبور خام کاربر که بک‌اند آن را هش می‌کند | حداقل ۸ و حداکثر ۱۰۰ کاراکتر |
| `isCompleteProfile` | `boolean` | خیر | در حال حاضر بک‌اند مقدار پروفایل کامل را هنگام ثبت‌نام `false` ذخیره می‌کند | پیشنهاد فرانت: `false` ارسال شود |
| `avatarImageName` | `string \| null` | خیر | نام فایل آواتار، در صورت وجود | می‌تواند `null` باشد |
| `gender` | `number` | بله | جنسیت کاربر | `1` = مرد، `2` = زن |
| `birthDate` | `string` | بله | تاریخ تولد با فرمت ISO 8601 | باید قبل از زمان فعلی باشد |

## Response موفق

HTTP Status: `200 OK`

```json
{
  "isSuccess": true,
  "message": "ثبت نام با موفقیت انجام شد",
  "data": {
    "userId": "9f6f47d3-6c8d-49b3-9a7c-d625bcb2c6c0",
    "role": "NormalUser"
  }
}
```

### فیلدهای خروجی موفق

| فیلد | نوع | توضیح |
| --- | --- | --- |
| `isSuccess` | `boolean` | وضعیت موفقیت عملیات |
| `message` | `string` | پیام قابل نمایش به کاربر |
| `data.userId` | `string` | شناسه کاربر ثبت‌شده |
| `data.role` | `string` | نقش پیش‌فرض کاربر؛ مقدار فعلی `NormalUser` است |

## Response ناموفق

HTTP Status: `200 OK`

```json
{
  "isSuccess": false,
  "message": "کاربری با این شماره موبایل قبلاً ثبت شده است",
  "data": null
}
```

### خطاهای قابل انتظار

| سناریو | پیام نمونه |
| --- | --- |
| نام خالی باشد | `نام الزامی است` |
| نام بیشتر از ۱۰۰ کاراکتر باشد | `نام نباید بیشتر از ۱۰۰ کاراکتر باشد` |
| نام خانوادگی خالی باشد | `نام خانوادگی الزامی است` |
| نام خانوادگی بیشتر از ۱۰۰ کاراکتر باشد | `نام خانوادگی نباید بیشتر از ۱۰۰ کاراکتر باشد` |
| شماره موبایل خالی باشد | `شماره موبایل الزامی است` |
| شماره موبایل معتبر نباشد | `شماره موبایل معتبر نیست` |
| رمز عبور خالی باشد | `رمز عبور الزامی است` |
| رمز عبور کمتر از ۸ کاراکتر باشد | `رمز عبور باید حداقل ۸ کاراکتر باشد` |
| رمز عبور بیشتر از ۱۰۰ کاراکتر باشد | `رمز عبور نباید بیشتر از ۱۰۰ کاراکتر باشد` |
| تاریخ تولد معتبر نباشد | `تاریخ تولد معتبر نیست` |
| جنسیت معتبر نباشد | `جنسیت معتبر نیست` |
| شماره موبایل قبلاً ثبت شده باشد | `کاربری با این شماره موبایل قبلاً ثبت شده است` |

> نکته مهم برای فرانت: حتی در خطاهای اعتبارسنجی، پاسخ با Status Code برابر `200 OK` برمی‌گردد؛ بنابراین تصمیم‌گیری UI باید بر اساس `isSuccess` انجام شود، نه فقط Status Code.

## نمونه استفاده با fetch

```ts
const response = await fetch(`${API_BASE_URL}/api/Auth`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    firstName: "علی",
    lastName: "رضایی",
    phoneNumber: "09123456789",
    passwordHash: "StrongPass123",
    isCompleteProfile: false,
    avatarImageName: null,
    gender: 1,
    birthDate: "1995-05-20T00:00:00",
  }),
});

const result = await response.json();

if (result.isSuccess) {
  // ثبت‌نام موفق است؛ در صورت نیاز کاربر را به صفحه لاگین هدایت کنید.
  console.log(result.data.userId, result.data.role);
} else {
  // پیام خطا را به کاربر نمایش دهید.
  console.error(result.message);
}
```

## چک‌لیست پیاده‌سازی فرانت

- شماره موبایل را قبل از ارسال با الگوی `^09\d{9}$` اعتبارسنجی کنید.
- رمز عبور را با نام فیلد `passwordHash` ارسال کنید؛ با وجود نام فیلد، مقدار ارسالی رمز عبور خام است و در بک‌اند هش می‌شود.
- مقدار `gender` را عددی ارسال کنید، نه رشته‌ای.
- تاریخ تولد را با فرمت قابل تبدیل به `DateTime` ارسال کنید؛ فرمت پیشنهادی ISO 8601 است.
- خطاها را از فیلد `message` بخوانید.
- پس از ثبت‌نام موفق، در صورت نیاز کاربر را به لاگین هدایت کنید، چون پاسخ ثبت‌نام شامل access token نیست.
