# مستند فرانت: رزرو، تایید حضور بیمار و بررسی منشی

## خلاصه بیزینس
- مشاور هنگام ثبت رزرو باید شهر بیمار، درصد احتمال حضور و توضیح پیش‌بینی حضور را ثبت کند.
- وقتی زمان رزرو برسد، فرانت باید رزروهای موعددار را به مشاور نشان بدهد تا حضور/عدم حضور بیمار را تایید کند.
- تایید مشاور نهایی نیست؛ منشی باید تایید کند که ادعای مشاور درست بوده یا نه.
- بعد از بررسی منشی، سیستم به صورت خودکار امتیاز مشاور را اضافه یا کم می‌کند.

> نکته مهم: در این تسک بک‌اند migration ساخته نشده است. قبل از استفاده واقعی باید ستون‌های جدید دیتابیس با فرآیند دیتابیسی پروژه آماده شوند.

## ثبت رزرو توسط مشاور
`POST /api/Reservation`

### فیلدهای جدید اجباری
```json
{
  "leadAssignmentId": 10,
  "consultantProfileId": 3,
  "reservationAt": "2026-06-27T10:30:00",
  "patientCity": "تهران",
  "attendanceProbabilityPercent": 80,
  "attendancePrediction": "بیمار گفت روز و ساعت رزرو شده داخل مطب حاضر می‌شود.",
  "description": "توضیح اختیاری"
}
```

### اعتبارسنجی‌ها
- `patientCity` اجباری است.
- `attendanceProbabilityPercent` باید بین 0 تا 100 باشد.
- `attendancePrediction` اجباری است.
- `reservationAt` باید آینده باشد.

## نمایش نوتیفیکیشن/لیست رزروهای موعددار برای مشاور
`GET /api/Reservation/DueConfirmations?consultantProfileId=3`

خروجی لیستی از رزروهایی است که زمان آن‌ها رسیده و هنوز مشاور حضور بیمار را تایید نکرده است. فرانت می‌تواند این endpoint را polling کند یا در صفحه مشاور به شکل کارت/نوتیفیکیشن نمایش بدهد.

## تایید حضور توسط مشاور
`POST /api/Reservation/ConfirmAttendance`

```json
{
  "reservationId": 25,
  "consultantProfileId": 3,
  "patientAttended": true,
  "note": "بیمار در مطب حاضر شد."
}
```

بعد از ثبت، وضعیت به یکی از این موارد تغییر می‌کند:
- `ConsultantConfirmedPresent`
- `ConsultantConfirmedAbsent`

## تسک فرانت برای داشبورد منشی
فعلا داشبورد منشی نداریم، اما باید یک بخش جدید برای بررسی تاییدهای مشاور طراحی شود:

### بررسی تایید مشاور توسط منشی
`POST /api/Reservation/ReviewAttendance`

```json
{
  "reservationId": 25,
  "secretaryUserId": "00000000-0000-0000-0000-000000000000",
  "approved": true,
  "note": "با حضور بیمار در مطب تطبیق داده شد."
}
```

- اگر `approved=true` باشد، سیستم امتیاز مثبت ثبت می‌کند.
- اگر `approved=false` باشد، سیستم امتیاز منفی ثبت می‌کند.

## وضعیت‌ها
فرانت مقدار `attendanceConfirmationStatus` را دریافت می‌کند:

| عدد | نام | معنی |
| --- | --- | --- |
| 1 | PendingConsultantConfirmation | منتظر تایید مشاور |
| 2 | ConsultantConfirmedPresent | مشاور گفته بیمار آمده |
| 3 | ConsultantConfirmedAbsent | مشاور گفته بیمار نیامده |
| 4 | SecretaryApproved | منشی تایید مشاور را قبول کرده |
| 5 | SecretaryRejected | منشی تایید مشاور را رد کرده |

## لیست شماره لید لحظه‌ای برای تست
برای امکان تست فرآیند شماره‌های لحظه‌ای، فرانت باید یک بخش تستی/ادمین اضافه کند که شماره لیدهای لحظه‌ای را نمایش دهد و امکان انتخاب یک شماره برای ادامه فرآیند تماس/رزرو را بدهد. این بخش فعلا می‌تواند کنار صفحه لیدها قرار بگیرد تا بدون نیاز به داشبورد منشی، تست سناریوی رزرو و تایید حضور انجام شود.
